# 网络安全等级保护测评高风险判定实施指引（试行）

## 1. Skill 目标
本 Skill 内置了《网络安全等级保护测评高风险判定实施指引（试行）》中的所有高风险判例规则。它能够根据您描述的安全问题、系统等级、行业属性及现有缓解措施，自动判定该问题是否构成高风险，并给出详细依据和整改建议。

## 2. 适用场景
- 等保测评现场问题快速判断
- 安全自查结果的高风险识别
- 安全整改优先级排序

## 3. 输入要求
请按以下格式提供待评估信息：
- **系统等级**：（第二级 / 第三级 / 第四级）
- **问题所属层面**：（例如：安全物理环境、安全通信网络、安全计算环境、安全管理中心、安全管理制度、安全管理机构、安全管理人员、安全建设管理、安全运维管理，或扩展要求层面如云计算、移动互联、物联网、工业控制系统）
- **具体问题描述**：详细描述现象
- **是否关键行业**：（是 / 否）
- **现有缓解措施**：（描述已有补偿控制，如无则填“无”）

## 4. 内置判定规则（核心逻辑）

本 Skill 依据文档第5.2节判定原则，结合各判例的“适用范围”“问题描述”和“风险评价”提炼出以下规则。每条规则均包含：**判例编号**、**名称**、**适用等级**、**触发条件**、**可缓解条件**（若有）。

### 4.1 安全通用要求判例规则

#### A. 安全物理环境
- **6.1.1.1 机房出入口访问控制措施缺失**  
  适用：三级及以上。  
  触发：机房出入口无任何访问控制（无门禁、无值守）。  
  可缓解：大楼受控 + 专人值守视频监控 → 可降级。

- **6.1.2.1 机房防火措施缺失**  
  适用：三级及以上。  
  触发：无消防系统或灭火设备失效/不合规。  
  可缓解：专人值守或视频监控 → 可降级。

- **6.1.3.1 机房短期备用电力供应措施缺失**  
  适用：三级及以上且业务连续性要求高。  
  触发：无UPS等或无法满足短期运行。  
  可缓解：多路供电 → 可降级。

- **6.1.3.2 机房应急供电措施缺失**  
  适用：四级且业务连续性要求高。  
  触发：无应急供电设施或不可用。  
  可缓解：多数据中心+应用级灾备 → 可降级。

#### B. 安全通信网络
- **6.2.1.1 网络设备业务处理能力不足**  
  适用：三级及以上且业务连续性要求高。  
  触发：关键设备性能无法满足高峰期。  
  缓解：无 → 不可降级。

- **6.2.1.2 重要网络区域边界访问控制措施缺失**  
  适用：二级及以上。  
  触发：重要区域部署在边界，或与其他区域无隔离措施。  
  可缓解：所有服务器启用主机访问控制 → 可降级（仅针对无隔离措施）。

- **6.2.1.3 关键线路和设备冗余措施缺失**  
  适用：三级及以上且业务连续性要求高。  
  触发：无冗余。  
  可缓解：多数据中心+应用级灾备 → 可降级。

- **6.2.2.1 采用无加密措施的网络通道传输重要数据**  
  适用：三级及以上。  
  触发：明文传输鉴别数据、敏感个人信息或重要业务数据。  
  可缓解：数据自身加密 或 应用层加密协议（如HTTPS）→ 可降级。

#### C. 安全区域边界
- **6.3.1.1 网络边界接入设备或端口不可控**  
  适用：二级及以上。  
  触发：未配置明确上联端口、IP/路由不明确、未关闭不使用的端口。  
  可缓解：边界部署严格访问控制策略（IP/端口级）→ 可降级。

- **6.3.1.2 无线网络管控措施缺失**  
  适用：三级及以上。  
  触发：重要网络与无线网络互联，且无受控边界设备或策略不当。  
  可缓解：无线设备/用户强管控（IP/MAC绑定）或强身份认证 → 可降级。

- **6.3.2.1 重要网络区域边界访问控制措施缺失或配置不当**  
  适用：二级及以上。  
  触发：区域边界缺少访问控制设备或策略失效。  
  可缓解：所有服务器启用主机访问控制 → 可降级。

- **6.3.3.1 外部网络攻击防御措施缺失**  
  适用：二级及以上。  
  触发：关键网络节点无检测/防护手段，或策略库半年未更新。  
  可缓解：所有主机部署入侵防范产品且库更新及时 → 可降级。

- **6.3.3.2 内部网络攻击防御措施缺失**  
  适用：三级及以上。  
  触发：关键节点无检测/防护手段，或策略库半年未更新。  
  可缓解：所有主机部署入侵防范产品且库更新及时 → 可降级。

- **6.3.3.3 网络行为分析措施缺失**  
  适用：关键行业三级及以上。  
  触发：无抗APT、全流量分析等，或威胁情报库/模型半年未更新。  
  可缓解：物理隔离+强逻辑隔离+物理访问控制+强准入 → 可降级。

#### D. 安全计算环境
- **6.4.1.1 存在空口令、弱口令、通用口令或无身份鉴别措施**  
  适用：二级及以上。  
  触发：存在空口令、弱口令（长度<6、常见字典）、通用口令。  
  可缓解：仅限本地登录/登录限制 或 双因素认证 → 可降级。

- **6.4.1.2 鉴别信息明文传输**  
  适用：二级及以上。  
  触发：远程管理时鉴别信息明文传输（含编码可还原）。  
  可缓解：双因素认证（仅获得口令无法登录）→ 可降级。

- **6.4.1.3 未采用两种或两种以上组合身份鉴别技术**  
  适用：三级及以上。  
  触发：未使用双因素（口令+密码技术/生物等）。  
  可缓解：多次动态鉴别、分阶段不同鉴别、仅限本地/带外管理 → 可降级。

- **6.4.2.1 未重命名默认账户名且未修改默认口令**  
  适用：二级及以上。  
  触发：存在公开默认账户/默认口令。  
  可缓解：双因素认证 或 登录限制（无法仅凭默认口令登录）→ 可降级。

- **6.4.2.2 访问控制策略存在缺陷或不完善，存在越权访问可能**  
  适用：三级及以上。  
  触发：应用/数据库/操作系统访问控制缺陷可致越权。  
  可缓解：本地访问/登录限制 + 操作审计 → 可降级。

- **6.4.3.1 开启多余的系统服务、默认共享和高危端口**  
  适用：二级及以上。  
  触发：开启多余服务/共享/高危端口且可远程访问。  
  可缓解：登录限制使漏洞难利用，或主机入侵/恶意代码防护有效 → 可降级。

- **6.4.3.2 存在可被利用的高危安全漏洞**  
  适用：二级及以上。  
  触发：通过互联网管理/访问的系统存在已披露高危漏洞；或非互联网系统经验证存在高危漏洞。  
  可缓解：对于非互联网系统，如有登录限制或主机防护使漏洞难利用 → 可降级（互联网系统无缓解，不可降级）。

- **6.4.3.3 数据有效性检验功能缺失或不完善**  
  适用：二级及以上。  
  触发：存在SQL注入、XSS、文件上传等高风险漏洞。  
  可缓解：若不对互联网开放且有登录限制+审计，或部署WAF等 → 可降级。

- **6.4.4.1 恶意代码防范措施缺失**  
  适用：二级及以上。  
  触发：未安装防恶意代码软件、授权过期或特征库超1个月未更新。  
  可缓解：若为Linux/Unix等非Windows且不对互联网开放，有USB管控/白名单/入侵检测等，或在网络层有恶意代码检测 → 可降级（Windows无缓解不可降级）。

- **6.4.5.1 重要数据明文传输**  
  适用：三级及以上。  
  触发：鉴别数据、重要个人信息或业务数据明文传输。  
  可缓解：对于鉴别数据，若双因素认证使获得鉴别信息无法登录 → 可降级；其他重要数据无缓解不可降级。

- **6.4.5.2 重要数据明文存储**  
  适用：三级及以上。  
  触发：鉴别数据、重要个人信息或业务数据明文存储。  
  可缓解：若不对互联网开放且仅限本地/带外管理，或有数据库防火墙/防泄露措施 → 可降级。

- **6.4.6.1 重要数据无本地备份措施**  
  适用：三级及以上。  
  触发：未提供备份，或备份到互联网网盘等。  
  可缓解：多数据中心/冗余部署使数据有多个可用副本 → 可降级（备份到网盘不可降级）。

- **6.4.6.2 数据处理系统冗余措施缺失**  
  适用：三级及以上且业务连续性要求高。  
  触发：重要数据处理设备未采用热冗余。  
  缓解：无 → 不可降级。

- **6.4.6.3 无异地灾备措施**  
  适用：四级且业务连续性要求高。  
  触发：未设立异地应用级容灾或无法切换。  
  缓解：无 → 不可降级。

- **6.4.7.1 违规采集和存储个人信息**  
  适用：二级及以上。  
  触发：未授权采集/存储违规个人信息。  
  缓解：无 → 不可降级。

- **6.4.7.2 违规访问和使用个人信息**  
  适用：二级及以上。  
  触发：未授权使用、泄露、未脱敏用于测试等。  
  缓解：无 → 不可降级。

#### E. 安全管理中心
- **6.5.1.1 远程管理路径安全防护措施不足**  
  适用：三级及以上。  
  触发：远程管理未采用安全传输路径。  
  缓解：无 → 不可降级。

- **6.5.1.2 审计记录存储时间不满足要求**  
  适用：三级及以上。  
  触发：审计记录留存不足6个月（法律法规要求）。  
  缓解：无 → 不可降级（新系统需评估能力）。

#### F. 安全管理制度、机构、人员、建设管理、运维管理
- **6.6.1.1 管理制度缺失**：三级及以上，无制度或制度不适用 → 不可降级。
- **6.7.1.1 未建立网络安全工作领导机构**：三级及以上，未成立 → 不可降级。
- **6.8.1.1 外部人员接入网络管理措施缺失**：二级及以上，无申请审批流程和记录 → 不可降级。
- **6.9.1.1 违规采购和使用网络安全产品**：二级及以上，违反国家规定采购 → 不可降级。
- **6.9.2.1 外包软件开发代码审计措施缺失**：三级及以上，未进行代码审计且开发方未提供第三方检测证明。  
  可缓解：定期自行审计且有制度要求，或合同明确安全责任+技术防控 → 可降级。
- **6.9.3.1 上线前未开展安全测试**：三级及以上，未测试或未整改高风险问题。  
  可缓解：若上线后定期检测且无高危风险 → 可降级（未整改不可降级）。
- **6.9.4.1 未定期进行等级测评**：三级及以上，未每年测评且高风险未整改 → 不可降级。
- **6.9.4.2 选择的测评机构不符合国家相关要求**：三级及以上，选择不合规或被通报机构 → 不可降级。
- **6.10.1.1 未制定重要事件应急预案**：二级及以上，无预案或内容不完整 → 不可降级。
- **6.10.1.2 未定期开展应急预案培训和演练**：三级及以上，未每年培训和演练 → 不可降级。

### 4.2 扩展要求判例规则（简要）

#### 云计算
- **7.1.1.1** 基础设施位于境外 → 不可降级。
- **7.2.1.1** 平台承载高于其等级的业务系统 → 不可降级。
- **7.2.1.2** 虚拟网络隔离缺失 → 不可降级。
- **7.2.1.3** 四级系统未划分独立资源池 → 不可降级。
- **7.2.1.4** 平台未提供安全防护能力 → 不可降级。
- **7.3.1.1** 虚拟化边界访问控制缺失：平台侧不可降级；客户侧若服务器启用主机控制可降级。
- **7.3.2.1** 平台对客户发起的攻击无检测或库更新超半年：若主机有入侵检测且更新及时可降级。
- **7.4.1.1** 镜像/快照未保护 → 不可降级。
- **7.4.2.1** 数据/个人信息违规出境 → 不可降级。
- **7.4.3.1** 客户未在本地保存备份：若备份至不同区域或不同平台可降级。
- **7.4.3.2** 未提供迁移技术手段 → 不可降级。
- **7.4.4.1** 删除数据未删除所有副本 → 不可降级。
- **7.5.1.1** 管理流量与业务流量未分离：若设置严格访问控制可降级。
- **7.6.1.1** 选择不合规服务商 → 不可降级。
- **7.6.2.1** 供应商选择不合规 → 不可降级。
- **7.7.1.1** 运维地点境外且未遵规定 → 不可降级。

#### 移动互联
- **8.1.1.1** 无线过度覆盖：若MAC过滤等准入控制可降级。
- **8.2.1.1** 有线无线边界无网关：若有设备管控或强认证可降级。
- **8.2.2.1** 无线接入无认证 → 不可降级。
- **8.2.3.1** 开启高风险功能（SSID广播、WEP等）：若MAC过滤或WIDS监测可降级。
- **8.2.3.2** 多AP同密钥 → 不可降级。
- **8.2.3.3** 无法检测无线攻击：若终端有管控软件可降级。
- **8.3.1.1** 安装未签名应用：若来源可靠或签名可信可降级。
- **8.3.1.2** 安装恶意软件：若采用管理流程审批可降级。
- **8.4.1.1** 未知渠道安装 → 不可降级。
- **8.4.1.2** 未经评估的开源软件 → 不可降级。

#### 物联网
- **9.1.1.1** 物理环境造成破坏 → 不可降级。
- **9.2.1.1** 非授权节点接入 → 不可降级。
- **9.3.1.1** 非授权用户配置：若物理环境有访问控制可降级。
- **9.3.2.1** 网关无身份鉴别：若有设备连接管控或物理控制可降级。

#### 工业控制系统
- **10.1.1.1** 室外控制设备防护缺失：若处于受控区域且设备本身具备防护可降级。
- **10.2.1.1** 工控与企业其他系统隔离缺失：若部署工业协议检测防火墙且严格策略可降级（仅针对未使用单向隔离的情况）。
- **10.2.1.2** 工控内部区域划分不当：若无访问控制措施且无缓解不可降级；若采用串口通信等严格传输控制可降级。
- **10.2.2.1** 广域网传输控制指令无加密：若对通信设备采用多因子鉴别+地址限定，或广域网有设备接入认证可降级。
- **10.3.1.1** 工控与企业系统间未限制通用服务（Email、Web等）→ 不可降级。
- **10.3.2.1** 无线设备管控缺失：若有接入管控和身份鉴别可降级。
- **10.4.1.1** 控制设备未更新：若网络边界严格隔离使漏洞难利用可降级（针对未更新）；若未使用专用设备更新则不可降级。
- **10.4.1.2** 控制设备上线前未检测恶意代码：若上线后定期检测且整改可降级（未整改不可降级）。
- **10.5.1.1** 工控重要设备未开展安全检测：若上线后定期检测且整改可降级（未整改不可降级）。

## 5. 判定工作流
当您输入问题时，本 Skill 将按以下步骤处理：
1. **解析输入**：识别系统等级、问题层面、问题描述、行业属性、缓解措施。
2. **规则匹配**：根据“问题描述”关键词和“等级”在以上规则库中查找最匹配的判例。
3. **条件判断**：
   - 检查等级是否满足“适用等级”。
